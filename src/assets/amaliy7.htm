7-amaliy mashg’ulot: Windows NT/2003/2008 tarmoq operatsion tizimlarida klient-server tarmog‘ini tashkil etish, ishchi stansiyalarda so‘rovlar shakllantirish, so‘rovlarni qayta ishlash funksiyalarini sozlash.

Ishdan maqsad: Windows Server 2008 ni administrlash foydalanuvchi va guruh xususiyatlarini yaratish va tashkil etish protseduralari bilan tanishishdir. Tizimli siyosat va audit bilan tanishishdir NTFS fayl tizimi tarmoq xossalarini ko’rib chiqish. Tarmoq va lokal resurslarga murojaatni boshaarishga o’rganishdir.

Amaliy ishni bajarishda zaruriy vositalar va axborot manbalari ta’minoti:

·        Har bir tinglovchi uchun ishchi stantsiya ajratilgan kompyuter sinfi;    

·        Mahalliy (lokal) hisoblash tarmog’i;

·        O’rnatilgan Windows 2008 Server operatsion tizimi  elementlari  yig’ilgan disk.

Nazariy qism

Windows Server 2008 operatsion tizimida klient-server tarmog‘ini hosil qilish va ularning qo‘shimcha vazifalarini qo‘shish uchun Add Roles ni bosamiz. Bu erda, serverimizga biz uchun kerak bo‘lgan xizmatlarni (Service)  yuklaymiz. Bu erda xizmatlarni rol deb nomlangan. Bu xizmatlarni ko‘plari oldingi Windows Server 2008 OTdagilarni xuddi o‘zi. Oldingi xizmatlardan tashqari bir qancha yangilari ham bor.

Bu erda serverimizga yuklanadigan rollar tanlanadi. Biz bu serverni domen kontroller qilib sozlamoqchimiz, shuning uchun Active Directory Domain Services ni tanlab  Next ni bosamiz.

Bu erda Active Directory Domain Services  yuklanib bo‘lganidan so‘ng serverni to‘liq domen kontroller qilish uchun yana dcpromo utilitasini ishga tushirishimiz lozim deb yozilgan. Install ni bosamiz va yuklanish jarayonini biroz kutamiz.

Oldingi vizard yopilib boshqasi ochildi. Bu erda Next ni bosamiz.

Bu mashg‘ulotimizda biz yangi o‘rmonda, yangi domen tuzayapmiz shuning uchun, Sreate a new domain In a new forest ni tanlaymiz va Next  ni bosamiz.

Endi yangi tuzayatgan domenimizni to‘liq nominini kiritamiz. FQDN (domenni to‘liq nomi) joyiga misol uchun virtualdars.uz deb yozamiz va Next ni bosamiz. SHunda vizard tarmog‘imizda bunday nomli domenni mavjudligini tekshirib chiqadi.

 Vizard bizdan yangi o‘rmonimizni funksional darajasini belgilashimizni so‘ramoqda. Sodda qilib aytganda, tarmog‘imizdagi mavjud domen kontrollerlar qaysi turdagi OTda ishlashi nazarda tutilmoqda. Bu erdan Windows Server 2008 ni tanlaymiz chunki biz mutlaqo yangi o‘rmon va yangi domen tuzayapmiz. Lekin agar ushbu tuzilayatgan yangi o‘rmonimizga keyinroq boshqa domen kontroller qo‘shmoqchi bo‘lsak u faqatgina Server 2008 Otda ishlovchi kontroller bo‘lishi lozim. Next ni bosamiz.

Bu yerda domen kontrollerimizni qo‘shimcha sozlashlari ko‘rsatilmoqda. Albatta DNS-server domen kontroller ishlashi uchun kerak buladi. Xozircha bizni serverda DNS xizmati yuklanmagan shuning uchun uni qoldirib Next ni bosamiz. Bu erda Active Directory ma’lumotlar bazasi, log-fayllar va Sysvol papkalarini joylashishi ko‘rsatilmoqda. Bularni shundayligicha qoldirib Next ni bosamiz.

Mana yana boyagi ICT oynasi ochildi. Ahamiyat beradigan bo‘lsak, serverimizni rollaridan Active Directory Domain Services, DNS Server va bularga qo‘shimcha qilib Group Policy Management va Remote Server Administration Tools lar ham yuklandi. Keling xozir uchinchi bo‘limni oxirigacha davom ettirib qo‘yamiz va Remote Desktopni yoqib (Enable) qo‘yamiz.

Xozir biz ishlayatgan server o‘quv tarmog‘ida bo‘lgani uchun Allow connections from computers running any version of Remote Desktop ni tanlaymiz. Albatta xavfsizlik tomonidan uchinchisini tanlagan ma’qulroq bo‘ladi. Select Users… ni bosib Remote Desktop orqali qaysi foydalanuvchilar serverimizga kirishi mumkinligini tanlasak bo‘ladi. Buni keyinchalik sozlab qo‘yish ham mumkin. OK ni bosamiz. Firewall ishlab turibdi. Boshlang‘ich sozlashlarni bajarib bo‘ldik. ICT oynasini yopamiz.

Endi serverni boshqarish uchun Quick Launchda joylashgan Server Manager dasturidan foydalanishimiz mumkin.

1. Foydalanuvchilarni tashkil etish tarmoq modellari: Ishchi guruh modeli va domen modeli.

2. Windows Server 2008 tipik domen modeli haqida.

3. qayd yozuvlarini administrlash (ma'murlash). Lokal va global qayd yozuvlari. Kiritilgan qayd yozuvi xususiyatlari.

4. Foydalanuvchilar guruhi. Global va lokal guruhlar. Maxsus guruhlar.

5. Qayd yozuvi va guruh imtiyozlari. Kiritilgan lokal guruh xossalari.
